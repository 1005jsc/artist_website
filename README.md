## 개발과정

1. 미술작가 웹사이트를 만들기로 결심

2. 어떤 니즈를 충족시킬 것인가?

    현  (좀 길 듯)

2. 웹사이트의 핵심 키워드를 '진정성'으로 정함
	
    a. 창작물이 진실되야할 것 

	b. 퀄리티를 가장 우선시 할 것

    c. 쉽고 간편한 유저 인터페이스를 구축할 것

    e. 작가의 신비성이 보장될 수 있도록 할 것
    

    f. 위 4가지 철학을 적용 해 본 결과 내 웹사이트는 이런 기준으로 만들어야 겠다라는 게 아래와 같이나옴

    g. 작가, 작가의 작품이 유저에게 가장 잘 인식될 수 있게 만들것
    
    h. 유저는 작가의 작품을 가장 좋은 화질, 편한 인터페이스로 볼 수 있게 만들 것 등의 원칙이 나옴

3. 어떤 

4. 어떤 기술들을 사용할 것인지 정함 react, typescript, create-react-app 
5. 웹사이트의 전체적인 사이즈 및 규격 설정: 크롬브라우저, 윈도우 환경을 기준으로 전체화면의 규격, 웹사이트 제작의 편의성들을 고려하여 가로 세로 비율 2대1로 정함 
6. 기초적인 웹디자인 공부(grid, 색상 정도 물론 전문적인 디자이너 실력은 절대 아님)
7. A4 4~50장 정도의 페이지와 컴포넌트의 Rough한 디자인을 그림
7. Figma로 대략적인 웹사이트의 디자인을 함
8. 제작시 적용할 html 사이즈 및 rem 기준 정하기 : html: 62.5%, 
코딩할 동안에는 body의 min-width: 1190px. Width: 1440px로 만들어 하고,
배포할 때는 전체적인 폰트가 vw를 기준으로 변하게끔 만듬 
9. 정한 디자인을 기준으로 컴포넌트들의 html, css 들을 작성 
10. App에 React-router로  경로를 정해 주고 button에 navigate경로를 정해줌
11. 로그인, 이미지 저장, 실시간 데이터베이스 등은 해당 api들을 제공해주는 업체들을 통해 실현 (firebase, cloudinary 둘다 문서화가 잘 되어있고 무료임)
12. 데이터 타입 설정
13. 자바스크립트적인 기능 구현(여기부분을 노션, 깃허브에 ㄷ)
14. Netlify로 배포 
결과물 평가 및 반성
